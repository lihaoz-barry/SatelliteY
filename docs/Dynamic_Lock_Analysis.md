# Windows Dynamic Lock 功能分析与建议

> 📅 创建日期：2026-01-02  
> 📌 适用场景：自动化工作流 + 安全性平衡

---

## 📋 功能概述

**Dynamic Lock** 是 Windows 的一个安全功能：
- ✅ 当配对的蓝牙设备（如手机）离开范围时，**自动锁定**电脑
- ❌ **不支持自动解锁**（设备回到范围内不会自动解锁）

> [!IMPORTANT]
> Dynamic Lock 只能 **锁定**，不能 **解锁**！
> 这意味着它不能完全解决你的「自动化工作流」问题。

## 🔍 成熟度评估

### 用户反馈总结（2024-2025）

| 维度 | 评分 | 说明 |
|------|------|------|
| **可靠性** | ⭐⭐☆☆☆ | 蓝牙连接不稳定导致误锁/不锁 |
| **兼容性** | ⭐⭐☆☆☆ | 与 Phone Link 应用冲突严重 |
| **延迟** | ⭐⭐⭐☆☆ | 离开后约 30-60 秒才锁定 |
| **配置** | ⭐⭐⭐⭐☆ | 设置简单，一键启用 |

### 常见问题

1. **与 Phone Link 冲突** - 最常见问题，需要禁用 Phone Link 自动启动
2. **蓝牙断连** - 驱动或省电设置导致假性"离开"
3. **更新后失效** - Windows 更新后需要重新配置
4. **不支持 iPhone** - Dynamic Lock 对 iPhone 支持较差（iOS 限制）

## 📱 你的设备分析

根据你的蓝牙设备列表：

| 设备 | 类型 | 推荐用于 Dynamic Lock |
|------|------|----------------------|
| **iPhone** | 手机 | ⚠️ 不推荐（iOS 限制蓝牙后台） |
| **WH-1000XM4** | 头戴耳机 | ❌ 不推荐（会摘下来） |
| **WF-1000XM4** | 耳塞 | ❌ 不推荐（会摘下来） |
| **MX Master 3** | 鼠标 | ❌ 不适用（固定在桌面） |
| **MX Keys** | 键盘 | ❌ 不适用（固定在桌面） |
| **Xbox Controller** | 游戏手柄 | ❌ 不适用（不随身携带） |
| **智能手表** | 穿戴设备 | ✅ **最佳选择**（如果有） |

> [!TIP]
> **智能手表** 是 Dynamic Lock 的最佳设备，因为：
> - 始终佩戴在身上
> - 蓝牙连接稳定
> - 不会像耳机一样摘下来

## ✅ 针对你场景的建议

### 你的需求分析

```
目标: 自动化工作流 + 安全性
     ├── 重启后自动登录 ✅ 已解决（netplwiz 自动登录）
     ├── 唤醒后自动登录 ✅ 已解决（禁用唤醒锁屏）
     └── 离开时自动锁定 ❓ Dynamic Lock（可选）
```

### 我的建议方案

#### 方案 A: 保持当前设置（推荐）

既然你已经配置了：
- ✅ 自动登录（重启后）
- ✅ 唤醒不锁屏

**不建议启用 Dynamic Lock**，原因：
1. iPhone 与 Dynamic Lock 兼容性差
2. 可能导致意外锁屏，破坏自动化工作流
3. 你说物理环境安全，不需要额外锁定

#### 方案 B: 如果仍想要安全性

如果你想在离开时自动锁定，建议：

1. **购买 Tile 追踪器或智能手表**（非 Apple Watch）
   - 蓝牙连接更稳定
   - 可以配对到 Windows

2. **手动锁定习惯**
   - 养成 `Win + L` 习惯
   - 比 Dynamic Lock 更可靠

3. **使用屏保锁定**
   - Settings → Personalization → Lock screen → Screen timeout
   - 设置 5-10 分钟无操作后锁定

## 🔧 如果你仍想尝试 Dynamic Lock

### 设置步骤

1. 确保手机已配对蓝牙到电脑
2. Settings → Accounts → Sign-in options
3. 勾选 **"Allow Windows to automatically lock your device when you're away"**

### 优化配置

```powershell
# 1. 禁用 Phone Link 自动启动（避免冲突）
# 打开 Phone Link → 设置 → 关闭 "Start Phone Link when I sign in"

# 2. 禁用蓝牙省电模式
# 设备管理器 → 蓝牙适配器 → 右键属性 → 电源管理
# 取消勾选 "Allow the computer to turn off this device to save power"
```

### iPhone 特别注意

由于 iOS 限制：
- iPhone 在后台时蓝牙信号可能不稳定
- 可能需要保持 Bluetooth 设置页面打开
- **不推荐 iPhone 用于 Dynamic Lock**

## 🎯 我的最终建议

对于你的个人电脑 + 自动化工作流场景：

```
✅ 保持当前设置（自动登录 + 唤醒不锁屏）
✅ 可选：设置 10-15 分钟屏保锁定
❌ 不建议启用 Dynamic Lock（与 iPhone 兼容性差，可能破坏工作流）
```

如果你真的需要「离开自动锁 + 回来自动解锁」的功能，**Windows 原生不支持**，需要第三方软件如：
- **Rohos Logon Key**（USB 设备解锁）
- **BLE Unlock**（蓝牙解锁，需要特定设备）

但这些都增加了复杂性，不如保持简单的自动登录方案。
